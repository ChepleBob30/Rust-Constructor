# Rust Constructor 资源

在`Rust Constructor`中，一切功能都以资源来实现，这些资源都实现了`Rust Constructor Resource`特征（以下简称`RCR`）。
因为各个资源间能做的事情以及体量的大小均有不同，它们被分成了三种类别：
  - 后端资源（对应`background.rs`）：负责处理一些简单的后台任务，并不承担渲染职责；
  - 基本前端资源（对应`basic_front.rs`）：负责显示简单的内容（例如文本，图片），且它们在创建时只会生成一个`RCR`；
  - 高级前端资源（对应`advance_front.rs`）：负责一些功能较为复杂的内容（例如开关），它们的显示需要依赖基本前端资源和后端资源。

# 资源结构

资源通常由`RustConstructorResourceBox`这一结构体包裹，存储于`App`结构体中的`rust_constructor_resource`。它包含名称与内容两个字段，其中名称由`RustConstructorId`结构体定义，包含名称与类型两个字段。

请注意，在较新的`Rust Constructor`版本中，类型不再需要手动书写，而是由创建资源的方法自动生成。你只需要记住，资源的类型等同于其结构体名称（例如图片资源的类型叫做`Image`）。

命名有以下两条硬性要求：
1. 资源名称必须是唯一的，不能与已有**同类型**资源冲突；
2. 资源名称不能留空。

除此以外，原则上命名没有限制。但我建议使用*帕斯卡命名法*，即每个单词的首字母大写，单词之间没有下划线或空格。

# 代码文件分类

`Rust Constructor`的代码文件分类基本按照资源分类进行，除去负责定义核心内容的`lib.rs`以及负责实现核心代码逻辑的`app.rs`。
