资源是一切功能的基石。你可以通过以下两种途径使用资源，不过它们的底层逻辑是相同的。

# 先添加后使用资源

这种方法较老，目前主要应用于高级前端资源的基本前端资源。

首先需要使用`add_resource`方法。该方法接收一个资源名称和一个资源内容。无论你使用什么资源，你都可以直接使用该方法添加资源并在后续使用。

创建资源内容时，你需要先找到你想要使用的资源结构体，然后以`xxx::default()`的形式创建资源内容。如果还需要进一步进行修改，你可以在`default()`后添加`.something(true)`之类的方法。

在此特别提醒，严禁直接在循环代码中添加资源！这么做很快就会造成资源重名错误，进而导致程序崩溃。

通常情况下，我们可以选择`check_resource_exists`方法。它只接收一个`RustConstructorId`，判断完毕后，如果该资源存在，则返回它的索引值，否则返回`None`。通过这种方法，我们可以在循环代码中安全地添加资源，避免资源重名错误。

还有一种方法是基于页面加载来实现，我们会在下文[添加页面](./add-pages.md)中介绍。

添加完毕后，你需要使用`use_resource`方法来使用已添加的资源。此方法接收一个`RustConstructorId`，一个用于渲染内容的`egui::Ui`，以及一个可选的`Box<dyn Config>`（这属于进阶内容，我会在下文进行讲解）。与添加资源相反，使用资源务必放在循环代码中，这样才能让你的资源始终处于运作状态。

# 快速放置

这种方法更加简便明了，如果没有特殊需求，推荐使用这种方法来使用资源。

你只需要使用`quick_place`方法即可。它所接受的内容几乎与`use_resource`一模一样，与`use_resource`不同的是，`quick_place`会在资源不存在时自动创建资源并在资源存在时调用它，这样一来你的工作量就大幅减少了。
