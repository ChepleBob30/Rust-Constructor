# Rust Constructor Resources

In `Rust Constructor`, every feature is implemented as a resource, and all resources implement the `Rust Constructor Resource` trait (hereafter `RCR`).
Since resources differ in functionality and scope, they are divided into three categories:
  - Backend resources (defined in `background.rs`): handle simple background tasks and do not participate in rendering;
  - Basic frontend resources (defined in `basic_front.rs`): display simple content (e.g., text, images) and produce only a single `RCR` upon creation;
  - Advanced frontend resources (defined in `advance_front.rs`): handle more complex functionality (e.g., switches) and depend on basic frontend resources and backend resources for rendering.

# Resource Structure

Resources are typically wrapped in the `RustConstructorResourceBox` struct and stored in the `rust_constructor_resource` field of the `App` struct. It contains two fields: name and content. The name is defined by the `RustConstructorId` struct, which contains two fields: name and type.

Note: in newer versions of `Rust Constructor`, the type is no longer entered manually; it is automatically generated by the resource creation method. Just remember that a resource's type equals its struct name (e.g., an image resource's type is `Image`).

Naming has two hard requirements:
1. The resource name must be unique and must not conflict with existing resources of the **same type**;
2. The resource name cannot be empty.

Beyond this, there are no naming restrictions in principle. However, I recommend using *PascalCase*: capitalize the first letter of each word with no underscores or spaces between words.

# Source File Classification

The source file organization in `Rust Constructor` largely follows the resource classification, except for `lib.rs` (which defines core types) and `app.rs` (which implements the core application logic).
